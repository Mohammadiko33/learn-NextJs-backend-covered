## 🎯 Dynamic Rendering یعنی چی؟

**Dynamic Rendering** یعنی صفحات وب به صورت **پویا و در لحظه** برای هر درخواست کاربر ساخته و ارسال میشن.
یعنی به محض اینکه کاربر درخواست میده، سرور محتوا رو با توجه به شرایط اون درخواست (مثلاً نوع دستگاه، کاربر خاص، داده‌های تازه) رندر و به صورت HTML ارسال می‌کنه.

---

## 🔍 تفاوت با Static Rendering

* در Static Rendering، صفحه فقط یکبار ساخته و ذخیره میشه و همیشه همون صفحه آماده به کاربر میرسه.
* در Dynamic Rendering، صفحه هر بار هنگام درخواست تازه ساخته میشه و می‌تونه اطلاعات به‌روز یا شخصی‌سازی شده باشه.

---

## 🛠 مثال ساده (مثلاً با Next.js)

فرض کن یک سایت فروش داری که موجودی محصولات به طور مداوم تغییر می‌کنه:

```tsx
// pages/product/[id].tsx

export async function getServerSideProps(context) {
  const { id } = context.params;
  // گرفتن داده محصول به صورت داینامیک
  const product = await getProductFromAPI(id);
  return {
    props: {
      product,
    },
  };
}

export default function ProductPage({ product }) {
  return (
    <div>
      <h1>{product.name}</h1>
      <p>قیمت: {product.price} تومان</p>
      <p>موجودی: {product.stock}</p>
    </div>
  );
}
```

اینجا:

* با هر درخواست کاربر، تابع `getServerSideProps` اجرا میشه.
* اطلاعات محصول جدید و دقیق از API گرفته میشه.
* صفحه به صورت پویا ساخته و ارسال میشه.

---

## 📦 مزایا

* اطلاعات همیشه به‌روز هستن و میشه به راحتی صفحه رو شخصی‌سازی کرد.
* مناسب برای سایت‌هایی با داده متغیر مثل فروشگاه آنلاین، داشبورد، شبکه‌های اجتماعی.

---

## 🚨 معایب

* زمان لود صفحه ممکنه بیشتر باشه چون سرور باید هر بار صفحه رو بسازه.
* بار سرور بالاتر میره چون هر درخواست نیازمند پردازش جداست.

---

## 🎯 خلاصه

Dynamic Rendering یعنی:

* صفحه هر بار هنگام درخواست ساخته میشه
* داده‌ها و محتوا به‌روز و قابل شخصی‌سازی هستن
* مناسب سایت‌های داده‌محور و پویا
